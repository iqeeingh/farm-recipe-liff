# Sheet Schema

## Core sheets

### `recipes`

Existing business sheet. This Apps Script expects at least:

- `recipeId`
- `recipeName`
- `productId`
- `productName`
- `category`
- `method`
- `sceneTags`
- `status`
- `sortOrder`

### `products`

Existing business sheet. This Apps Script expects at least:

- `productId`
- `productName`
- `status`
- `sortOrder`

### `recipe_events`

Raw append-only event log used by the LIFF frontend and maintenance jobs.

Columns:

1. `eventId`
2. `lineUserId`
3. `visitorId`
4. `eventType`
5. `method`
6. `keyword`
7. `category`
8. `productId`
9. `productName`
10. `recipeId`
11. `recipeName`
12. `createdAt`

Retention:

- Keep raw `recipe_events` for 5 years.
- `cleanupOldRecipeEvents()` deletes rows older than 5 years.
- Header row is always preserved.

## Derived sheets

These sheets are created by `setupDailyStatsSheets()` if missing.

### `daily_method_stats`

- `date`
- `method`
- `count`
- `updatedAt`

Source:

- counts only `click_method`

### `daily_recipe_stats`

- `date`
- `recipeId`
- `recipeName`
- `viewCount`
- `productClickCount`
- `updatedAt`

Source:

- `view_recipe`
- `click_product`

### `daily_product_stats`

- `date`
- `productId`
- `productName`
- `viewCount`
- `productClickCount`
- `updatedAt`

Source:

- `view_recipe`
- `click_product`

### `daily_scene_stats`

- `date`
- `sceneTag`
- `count`
- `updatedAt`

Source:

- `sceneTags` on recipes referenced by `view_recipe` and `click_product`

### `daily_event_stats`

- `date`
- `eventType`
- `count`
- `updatedAt`

Source:

- all event types in `recipe_events`

### `user_tags`

Generated by `rebuildUserTagsRecent(days)`.

- `userKey`
- `lineUserId`
- `visitorId`
- `tag`
- `score`
- `updatedAt`

Current tag families:

- `prefer_method_*`
- `interest_category_*`
- `interest_product_*`
- `scene_*`
- `intent_click_product`

## Maintenance behavior

### `summarizeRecipeEventsForDate(targetDate)`

- Summarizes one day of `recipe_events`
- Replaces that day’s rows in each `daily_*_stats` sheet
- Safe to rerun without duplicate rows

### `summarizeYesterdayRecipeEvents()`

- Uses `Asia/Taipei`
- Summarizes yesterday only

### `dailyRecipeMaintenance()`

Runs:

1. `summarizeYesterdayRecipeEvents()`
2. `rebuildUserTagsRecent(180)`
3. `cleanupOldRecipeEvents()`

## Analytics API

The Apps Script web app also exposes read-only analytics endpoints backed only by the `daily_*_stats` sheets.

- `action=getAnalyticsSummary`
- `action=getDailyMethodStats`
- `action=getDailyRecipeStats`
- `action=getDailyProductStats`
- `action=getDailySceneStats`
- `action=getDailyEventStats`

Optional query parameter:

- `days=7`
- `days=30`
- `days=90`
- `days=180`
- `days=365`
